Sigma Rules

Sigma is an open standard for writing SIEM rules to detect suspicious or malicious behavior across platforms.

About Sigma

Sigma rules provide a generic way to describe detection logic for security events, making it easier to share and implement across different tools like Splunk, Elastic, or Azure Sentinel.

Sample Sigma Rule: Detect Failed Login Attempts

title: Failed Login Attempts
id: abc1234
status: experimental
description: Detects multiple failed login attempts in a short time
logsource:
  product: windows
  service: security
detection:
  selection:
    EventID: 4625
  condition: selection | count() > 5 within 5m
fields:
  - EventID
  - TargetUserName
  - SourceIpAddress
level: medium
