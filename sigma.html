Sigma Rules

Sigma is an open standard for writing generic detection logic for security events, enabling portability across various SIEM systems.

About Sigma

Sigma rules provide a flexible and vendor-agnostic way to describe threat detection logic. This allows security analysts to define suspicious behavior once and then convert it into native queries for different SIEMs (like Splunk, Elastic, Microsoft Sentinel, etc.). It's a critical component of modern detection engineering, promoting collaboration and consistency in threat intelligence.

Sample Sigma Rule: Detect Failed Login Attempts

This example demonstrates a basic Sigma rule designed to identify multiple failed login attempts on a Windows system, indicating a potential brute-force attack. 

title: Failed Login Attempts - Brute Force Detection
id: abc1234-5678-90ab-cdef-1234567890ab
status: experimental
description: Detects multiple failed login attempts from a single source within a short timeframe, indicative of a brute force attack.
author: Your Name/Handle
logsource:
  product: windows
  service: security
detection:
  selection:
    EventID: 4625 # Event ID for failed login attempts on Windows
  timeframe: 5m
  condition: selection | count() > 5 # More than 5 failed attempts in 5 minutes
fields:
  - EventID
  - TargetUserName
  - IpAddress # Or SourceIp if available
  - ComputerName
level: medium
tags:
  - attack.attempt.brute_force
  - attack.t1110 # MITRE ATT&CK: Brute Force

Note: Replace Your Name/Handle with your actual name or a suitable alias if contributing. The id should be a unique UUID.
